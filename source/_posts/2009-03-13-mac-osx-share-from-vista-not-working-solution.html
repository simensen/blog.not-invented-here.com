After trying to get my new Vista install connecting to my Mac's Samba share, I discovered that the commandline smbclient would actually connect just fine. A quick web search found this post that discussed a fix, and it appears that it works! This fix requires entering the Group Policy Editor ( Start > Run > gpedit.msc ) and navigating to "Security Options": -> Local Computer Policy -> Computer Configuration -> Windows Settings -> Security Settings -> Local Policies -> Security Options For the "Network security: LAN Manager authentication level" setting, change to "Send LM & NTLM - use NTLMv2 session security if negotiated." Worked like a charm!
